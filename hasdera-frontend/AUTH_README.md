# 🔐 מערכת התחברות והרשמה - מפרסמים בלבד

## ⚠️ חשוב: כל הברנץ' הזה מיועד למפרסמים בלבד!

## 📋 סקירה כללית

נוצרה מערכת התחברות והרשמה למפרסמים בלבד עם:
- דף התחברות/הרשמה מעוצב (מפרסמים בלבד)
- כניסה עם Google
- משתנים קבועים לכניסה מהירה בפיתוח
- ניתוב אוטומטי לדשבורד מפרסמים אחרי התחברות
- Protected Routes - כל הדפים מוגנים למפרסמים בלבד

## 🎯 קבצים שנוצרו/עודכנו

### קבצים חדשים:
1. **`src/contexts/AuthContext.jsx`** - Context לניהול מצב התחברות
2. **`src/Components/LoginPage.jsx`** - דף התחברות/הרשמה מעוצב
3. **`src/Components/PublicRoute.jsx`** - Route למשתמשים לא מחוברים
4. **`src/config/devCredentials.js`** - משתנים קבועים לכניסה בפיתוח

### קבצים שעודכנו:
1. **`src/App.jsx`** - הוספת AuthProvider ו-Protected Routes
2. **`src/Components/ProtectedRoute.jsx`** - שימוש ב-AuthContext
3. **`src/Services/Login.js`** - עדכון נתיבי API

## 🔑 משתנים קבועים לכניסה בפיתוח

בקובץ `src/config/devCredentials.js` יש משתמש מפרסמת קבוע:

```javascript
// מפרסמת
email: 'advertiser@hasdera.com'
password: 'advertiser123'
```

**⚠️ חשוב:** כפתורי הכניסה המהירה מופיעים רק בפיתוח (`import.meta.env.DEV`).

## 🛣️ ניתוב אחרי התחברות

אחרי התחברות מוצלחת, כל מפרסמת מנותבת אוטומטית ל:
- **Advertiser** → `/Navbar` (דשבורד מפרסמים)

משתמשים עם Role אחר לא יכולים להיכנס למערכת.

## 🔒 Protected Routes

**כל הדפים מוגנים למפרסמים בלבד:**

- **`/Navbar`** - מפרסמים בלבד
- **`/issues`** - מפרסמים בלבד
- **`/advertiser/placement`** - מפרסמים בלבד
- **`/advertiser/payment`** - מפרסמים בלבד
- **`/viewer`** - מפרסמים בלבד

משתמשים עם Role אחר יועברו אוטומטית ל-`/login`.

## 🚀 שימוש

### התחברות רגילה:
1. גשי ל-`/login`
2. הזיני מייל וסיסמה
3. לחצי "התחברות"

### הרשמה:
1. גשי ל-`/login`
2. לחצי על טאב "הרשמה"
3. מלאי את הפרטים (Role הוא תמיד "מפרסמת")
4. לחצי "הרשמה"

### כניסה עם Google:
1. גשי ל-`/login`
2. לחצי על כפתור Google
3. בחרי חשבון Google

### כניסה מהירה בפיתוח:
1. גשי ל-`/login`
2. בתחתית הדף תראי כפתור כניסה מהירה
3. לחצי על "מפרסמת" לכניסה מהירה

## 📝 הערות

- ⚠️ **כל הברנץ' הזה מיועד למפרסמים בלבד!**
- הטוקן נשמר ב-`localStorage` בשם `hasdera_token`
- פרטי המשתמש נשמרים ב-`localStorage` בשם `hasdera_user`
- אם הטוקן לא תקף (401), המשתמש מועבר אוטומטית ל-`/login`
- דף ההתחברות לא נגיש למפרסמים שכבר מחוברים
- משתמשים עם Role אחר לא יכולים להיכנס למערכת
- כשתאחדי את הברנצ'ים, תצטרכי להפריד בין מפרסמים למשתמשים אחרים
